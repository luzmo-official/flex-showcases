# Luzmo Analytics Components Kit (ACK)

ACK is a set of Lit-based web components for building **Luzmo chart configuration UIs** (drag & drop, pickers, option editors). ACK produces/edits chart configuration (slots/options/filters). Render the resulting configuration with **Luzmo Embed** (`@luzmo/embed` and its framework wrappers).

## Scope

This guide covers:
- Building chart configuration UIs with ACK panel + data components
- Generating and using **embed authorization** tokens (`authKey` / `authToken`)
- Data fetching via the built-in DataBroker
- Theming via global and component CSS variables
- Item definitions (chart types, slot config, options config)

## Start Here (End-to-End)

1. **Backend**: generate an **embed authorization** token (server-side) using your org API credentials.
2. **Frontend**: fetch your backend's embed authorization response and **read the token fields** (`authKey`/`authToken` *or* `id`/`token`, depending on what your backend returns) and pass them to ACK components via `auth-key` / `auth-token`.
3. **State**: treat `slotsContents`, `options`, and `filters` as *your app state* (single source of truth).
4. **Render**: pass `{ type, slots, options, filters }` into Luzmo Embed to render the configured chart/dashboard.

Minimal (vanilla web components) wiring sketch:

```ts
import '@luzmo/analytics-components-kit/item-data-drop-panel';
import '@luzmo/analytics-components-kit/edit-item';
import '@luzmo/embed'; // or a framework wrapper like @luzmo/react-embed

const auth = await fetch('/api/embed-token', { method: 'POST' }).then((r) => r.json());
// Your backend should ideally return { authKey, authToken }.
// If it returns the raw Luzmo authorization object instead, read { id, token } from it.
const authKey = auth.authKey ?? auth.id;
const authToken = auth.authToken ?? auth.token;

let slotsContents = [];
let options = {};
const itemType = 'bar-chart';

const dropPanel = document.querySelector('luzmo-item-data-drop-panel');
dropPanel.itemType = itemType;
dropPanel.authKey = authKey;
dropPanel.authToken = authToken;
dropPanel.slotsContents = slotsContents;

dropPanel.addEventListener('luzmo-slots-contents-changed', (e) => {
  slotsContents = e.detail.slotsContents;
  dropPanel.slotsContents = slotsContents; // re-apply state (one-way data flow)
});
```

### Key Components by Purpose

**Configuration UI (from @luzmo/analytics-components-kit)**:
- **luzmo-draggable-data-fields-panel** - Panel listing all draggable dataset columns/formulas for a dataset; optional dataset picker.
- **luzmo-item-data-drop-panel** - Panel rendering all droppable slots for a chart type; loads slot config from `item-type` and applies inter-slot constraints.
- **luzmo-item-data-picker-panel** - Panel rendering all slots as dropdowns (no drag & drop); optional dataset picker and constraints support.
- **luzmo-draggable-data-field** - Represents one data field (column/formula) for drag-and-drop.
- **luzmo-droppable-slot** - Drop zone representing a chart slot, receives draggable data fields.
- **luzmo-slot-contents-picker** - Combination of draggable data field and droppable slot into one dropdown-based component. Dropdown picker alternative for mobile/non-drag UI.
- **luzmo-edit-item** - Chart options editor (colors, labels, modes, etc.)
- **luzmo-edit-filters** - Chart filters editor. Supports nested groups and AND/OR filters groups.
- **luzmo-grid** - Editable dashboard grid layout manager

**Chart Rendering (from @luzmo/embed)**:
- **luzmo-embed-viz-item** - Renders a single configured chart
- **luzmo-dashboard** - Renders a full dashboard (if using pre-built dashboards)

**Data Flow**:
1. User interacts with ACK components (drag columns, edit options)
2. ACK components emit events with configuration updates
3. Your app updates state (slots, options)
4. Pass updated config to luzmo-embed-viz-item
5. Embedded chart re-renders with new configuration

## Auth & Data Access

**­¤öÆ CRITICAL SECURITY REQUIREMENT**: All Luzmo components (both ACK configuration UI and Luzmo Embed rendering) require authorization tokens to access data. Understanding this security model is essential before building your application.

**Terminology (important)**:
- **Org API credentials (server-side only)**: `LUZMO_API_KEY` + `LUZMO_API_TOKEN`. Never send these to the browser.
- **Embed authorization (sent to the browser)**: pass the embed authorization **key** and **token** to the browser. In the Luzmo API/SDK authorization object, these fields are named **`id` (auth key)** and **`token` (auth token)**. Your backend endpoint can return them as `{ authKey, authToken }` or return the raw `{ id, token }` object ŌĆö just make sure your frontend reads the right fields.

### ŌÜĀ’ĖÅ Golden Rules

1. **NEVER** expose your org API Key/Token (LUZMO_API_KEY / LUZMO_API_TOKEN) in frontend code
2. **ALWAYS** generate Embed Authorization tokens server-side
3. **ALWAYS** validate user identity before generating tokens
4. **ALWAYS** set appropriate token expiry (recommended: 24 hours max)
5. **ALWAYS** use filters for multi-tenancy to ensure data isolation

### Backend: Generate Embed Authorization Token

Your backend must create an endpoint that generates Embed Authorization tokens. This token is what your frontend will use.

**Node.js Example**:
```javascript
// backend/api/embed-token.js
import Luzmo from '@luzmo/nodejs-sdk';

const client = new Luzmo({
  api_key: process.env.LUZMO_API_KEY,      // ŌÜĀ’ĖÅ Server-side only!
  api_token: process.env.LUZMO_API_TOKEN,  // ŌÜĀ’ĖÅ Server-side only!
  host: 'https://api.luzmo.com'
});

// Express route example
app.post('/api/embed-token', async (req, res) => {
  // 1. Verify user is authenticated
  const user = req.user; // From your auth middleware
  if (!user) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  // 2. Generate Embed Authorization token
  const authorization = await client.create('authorization', {
    type: 'embed',
    username: user.id,           // Immutable user identifier
    name: user.name,
    email: user.email,
    suborganization: user.orgId, // For multi-tenancy
    expiry: new Date(Date.now() + 24 * 60 * 60 * 1000).toISOString(), // 24 hours
    inactivity_interval: 3600,   // 1 hour inactivity
    
    // 3. Specify access rights
    access: {
      datasets: [
        {
          id: 'your-dataset-id',
          rights: 'use'            // read, use, modify, own
        }
      ],
      dashboards: [
        {
          id: 'your-dashboard-id',
          rights: 'use'
        }
      ]
    },
    
    // 4. Apply multi-tenancy filters (CRITICAL for data isolation)
    filters: [
      {
        clause: 'where',
        origin: 'authorization',   // Applied to ALL queries
        column_id: 'customer_id',
        securable_id: 'your-dataset-id',
        expression: '? = ?',
        value: user.customerId     // User can only see their data
      }
    ]
  });

  // 5. Return ONLY the embed token (not API credentials!)
  res.json({
    authKey: authorization.id,
    authToken: authorization.token
  });
});
```

**Note**: The Luzmo authorization object returned by the API/SDK uses `id` and `token` fields (there is no `authKey` field on that object). If you return the raw object to the frontend, it must read `id`/`token`. If you map it server-side (recommended), your frontend can read `{ authKey, authToken }`.

### Frontend: Request and Use Token

Your frontend requests the token from your backend endpoint and uses it with all Luzmo components.

Example (framework-agnostic):

```ts
const authResponse = await fetch('/api/embed-token', {
  method: 'POST',
  credentials: 'include'
}).then((r) => r.json());

// Support either backend response shape:
// - { authKey, authToken } (recommended)
// - { id, token } (raw Luzmo authorization object)
const authKey = authResponse.authKey ?? authResponse.id;
const authToken = authResponse.authToken ?? authResponse.token;

// ACK components
dropPanel.authKey = authKey;
dropPanel.authToken = authToken;

// Luzmo Embed components
vizItem.authKey = authKey;
vizItem.authToken = authToken;
```

In React/Angular/Vue, bind `authKey`/`authToken` as component props / element properties (see [Installation & Frameworks](#installation--frameworks)).

### Multi-Tenancy with Filters

Authorization filters ensure each user/organization only sees their own data. There are two types:

**1. Authorization Filters** (Applied to ALL queries):
```javascript
{
  clause: 'where',           // or 'having'
  origin: 'authorization',   // Applied automatically
  column_id: 'customer_id',
  securable_id: 'dataset_id',
  expression: '? = ?',
  value: user.customerId     // From your auth system
}
```

**2. Initialization Filters** (Applied to specific charts):
```javascript
{
  clause: 'where',
  origin: 'initialization',  // Pre-fills filter UI
  chart_id: 'filter-chart-id',
  expression: '? = ?',
  value: 'USA'              // Can be changed by user
}
```

**Common Multi-Tenancy Patterns**:

```javascript
// Pattern 1: Single customer per user
filters: [
  {
    clause: 'where',
    origin: 'authorization',
    column_id: 'customer_id',
    securable_id: 'sales_dataset',
    expression: '? = ?',
    value: user.customerId
  }
]

// Pattern 2: Multiple customers per user
filters: [
  {
    clause: 'where',
    origin: 'authorization',
    column_id: 'customer_id',
    securable_id: 'sales_dataset',
    expression: '? in ?',
    value: user.allowedCustomerIds  // Array
  }
]

// Pattern 3: Date range restriction
filters: [
  {
    clause: 'where',
    origin: 'authorization',
    column_id: 'order_date',
    securable_id: 'sales_dataset',
    expression: '? >= ?',
    value: user.dataAccessStartDate
  }
]

// Pattern 4: Multiple filters (AND logic)
filters: [
  {
    clause: 'where',
    origin: 'authorization',
    column_id: 'customer_id',
    securable_id: 'sales_dataset',
    expression: '? = ?',
    value: user.customerId
  },
  {
    clause: 'where',
    origin: 'authorization',
    column_id: 'region',
    securable_id: 'sales_dataset',
    expression: '? in ?',
    value: user.allowedRegions
  }
]
```

### Token Expiry and Refresh

Tokens expire for security. Handle expiry gracefully:

```javascript
// Token refresh strategy
let currentAuth = null;
let tokenExpiryTime = null;

async function getValidToken() {
  // Check if token is still valid
  if (currentAuth && tokenExpiryTime && Date.now() < tokenExpiryTime - 60000) {
    return currentAuth;
  }

  // Request new token
  const response = await fetch('/api/embed-token', {
    method: 'POST',
    credentials: 'include'
  });
  
  const data = await response.json();
  currentAuth = {
    // data may be { authKey, authToken } OR raw { id, token }
    key: data.authKey ?? data.id,
    token: data.authToken ?? data.token
  };
  
  // Set expiry (assume 24 hours minus 1 minute buffer)
  tokenExpiryTime = Date.now() + (24 * 60 * 60 * 1000) - 60000;
  
  return currentAuth;
}

// Listen for expiry events from Luzmo components
dashboardComponent.addEventListener('authorization-expired', async () => {
  console.log('Token expired, refreshing...');
  
  const newAuth = await getValidToken();
  
  // Update all Luzmo components with new token
  dashboardComponent.authKey = newAuth.key;
  dashboardComponent.authToken = newAuth.token;
});
```

### Access Rights Levels

When creating tokens, specify appropriate rights:

- **`read`**: View-only access to dashboards; query datasets only in existing charts
- **`use`**: View dashboards + create variants; use datasets in chart configuration
- **`modify`**: Edit dashboards/datasets
- **`own`**: Full control including sharing and deletion

```javascript
// Example: Different rights for different resources
access: {
  datasets: [
    {
      id: 'public-dataset',
      rights: 'use'        // Can use in chart builder
    },
    {
      id: 'premium-dataset',
      rights: 'read'       // View-only in existing charts
    }
  ],
  dashboards: [
    {
      id: 'template-dashboard',
      rights: 'use'        // Can duplicate and modify variants
    },
    {
      id: 'shared-dashboard',
      rights: 'read'       // View-only
    }
  ]
}
```

### Security Checklist

Before going to production, verify:

- [ ] Org API Key/Token stored server-side (environment variables; never in client code)
- [ ] Token generation endpoint is authenticated
- [ ] Token generation is server-side only
- [ ] Appropriate filters applied for multi-tenancy
- [ ] Token expiry set (max 24 hours recommended)
- [ ] Token refresh logic implemented
- [ ] Access rights match user permissions
- [ ] HTTPS used for all requests
- [ ] CORS configured properly
- [ ] Error handling doesn't leak sensitive data

### Framework Notes (Properties vs Attributes)

**Important**: ACK components are web components. How you pass data depends on your framework:

- **Plain HTML**: attributes are strings; for objects/arrays, set element properties in JS (`el.slotsContents = [...]`) and use `addEventListener(...)`.
- **Lit templates**: bind element properties with a leading `.` and listen to events with `@` (Lit-specific syntax).
- **Angular**: bind properties with `[prop]` and events with `(event)` (for example `(luzmo-slots-contents-changed)="onChange($event)"`).
- **Vue**: bind properties with `:prop` and events with `@event`.
- **React**: prefer the generated wrappers (`@luzmo/analytics-components-kit/react`) so you can pass objects/arrays as typed props.

**Reading this doc**:
- `html` code blocks are copy/pasteable **plain HTML** (static string/number/boolean attributes only).
- For **objects/arrays** and **event handlers**, use the `ts` block directly below the HTML example (property assignment + `addEventListener(...)`).

## TypeScript Types

### TypeScript Types and Interfaces

The Analytics Components Kit exports TypeScript types and interfaces for event details and component properties.

**Importing Types**:
```typescript
// Import event detail types
import type { InfoVisibilityChangedEventDetail } from '@luzmo/analytics-components-kit/types';
import type { SlotContentsChangedEventDetail } from '@luzmo/analytics-components-kit/types';
import type { VizItemMenuButton } from '@luzmo/analytics-components-kit/types';

// Use in event handlers
function handleInfoVisibilityChanged(event: CustomEvent<InfoVisibilityChangedEventDetail>) {
  const { isVisible, slotContent } = event.detail;
  console.log('Info visible:', isVisible);
  console.log('Dataset ID:', slotContent.datasetId);
}
```

**Available Types**:
- `InfoVisibilityChangedEventDetail` - Event detail for info visibility changes
  ```typescript
  {
    isVisible: boolean;
    slotContent: GenericSlotContent;  // Contains columnId, datasetId, formula, level, etc.
  }
  ```
- `SlotContentsChangedEventDetail` - Event detail for slot content changes
- `VizItemMenuButton` - Union type for viz item menu button types ('edit' | 'clone' | 'delete')
- `DatasetDataField` - Data structure for draggable data fields
- `GenericSlotContent` - Structure for slot content items
- `DatasetDataField` - Discriminated union representing a selected data field. Variants:
  - `ColumnFieldData` - Column data field (`columnId`, `datasetId`, `type`, `name`, ...)
  - `FormulaFieldData` - Saved formula data field (`formulaId`, `datasetId`, `type`, `name`, ...)
  - `AdHocFormulaFieldData` - Ad hoc formula data field (`formula` expression string, `datasetId`, `type`, `name`, ...)
- `AdHocFormulaInput` - Input shape for ad hoc formulas in `datasetsDataFields[].formulas` (`formula`, `label?`, `type`, `subtype?`, `format?`)
- `ChatMessage` - AI chat message shape (`id?`, `type: 'user' | 'ai'`, `message: string`)
- `LuzmoChart` - Chart config passed to ai-chat for context (version, type, position, title, options, content, slots, filterGroups, aichartId)
- `AISampleChart` - AI API response shape (generatedChart, question, functionCall, functionResponse, feedback, message?, conversationId?)

## Component Reference

**Auth note**: `auth-key` / `auth-token` in component props refer to **embed authorization** tokens, not your org API credentials. In a raw Luzmo authorization object these fields are named `id` (auth key) and `token` (auth token).

### Data Panels

**Prefer data panels over individual data primitives.** Use panel components for chart slot configuration instead of the individual components (`luzmo-draggable-data-field`, `luzmo-droppable-slot`, `luzmo-slot-contents-picker`) unless you need custom styling/layout not possible with panels. Panels automatically handle slot constraints (e.g., "no multiple measures if legend is filled in a bar chart"). If using individual components directly, you must implement these constraints yourself.

#### Responsive Pattern: Combining Panels

Use drag & drop panels on desktop and picker panel on mobile. Both emit the same `luzmo-slots-contents-changed` event, so share the same state handler.

```html
<!-- Desktop: drag & drop experience -->
<div class="desktop-only" style="display: flex; gap: 1rem;">
  <luzmo-draggable-data-fields-panel
    draggable-data-field-variant="highlight"
    id="desktopFields">
  </luzmo-draggable-data-fields-panel>
  <luzmo-item-data-drop-panel
    id="desktopDropPanel"
    item-type="bar-chart"
    grows>
  </luzmo-item-data-drop-panel>
</div>

<!-- Mobile: picker experience -->
<div class="mobile-only">
  <luzmo-item-data-picker-panel
    id="mobilePickerPanel"
    item-type="bar-chart"
    grows
    selects="multiple">
  </luzmo-item-data-picker-panel>
</div>
```

```ts
const datasetIds = ['dataset-id-1', 'dataset-id-2'];
let slotsContents = []; // VizItemSlots (your app state)

const desktopFields = document.getElementById('desktopFields') as HTMLElementTagNameMap['luzmo-draggable-data-fields-panel'];
const desktopDropPanel = document.getElementById('desktopDropPanel') as HTMLElementTagNameMap['luzmo-item-data-drop-panel'];
const mobilePickerPanel = document.getElementById('mobilePickerPanel') as HTMLElementTagNameMap['luzmo-item-data-picker-panel'];

desktopFields.datasetIds = datasetIds;
desktopDropPanel.slotsContents = slotsContents;
mobilePickerPanel.datasetIds = datasetIds;
mobilePickerPanel.slotsContents = slotsContents;

const applySlots = (next: typeof slotsContents) => {
  slotsContents = next;
  desktopDropPanel.slotsContents = slotsContents;
  mobilePickerPanel.slotsContents = slotsContents;
};

desktopDropPanel.addEventListener('luzmo-slots-contents-changed', (e) => {
  applySlots((e as CustomEvent<{ slotsContents: typeof slotsContents }>).detail.slotsContents);
});
mobilePickerPanel.addEventListener('luzmo-slots-contents-changed', (e) => {
  applySlots((e as CustomEvent<{ slotsContents: typeof slotsContents }>).detail.slotsContents);
});
```

```css
/* Toggle visibility based on viewport */
@media (min-width: 1024px) {
  .desktop-only { display: flex; }
  .mobile-only { display: none; }
}
@media (max-width: 1023px) {
  .desktop-only { display: none; }
  .mobile-only { display: block; }
}
```

**Key points**:
- Both panels share the same `slotsContents` state
- Both emit `luzmo-slots-contents-changed` with identical `event.detail.slotsContents`
- Use CSS or JavaScript to show/hide based on viewport
- Keep `item-type` synchronized between both panels

**Finding Available Options and Keys**:

`option-key` must match a `key` entry in the options configuration for the selected `item-type`.

Use the exported modules (recommended) instead of browsing `node_modules` manually:

```ts
import { itemTypes } from '@luzmo/analytics-components-kit/item-list';
import { barChartOptionsConfig } from '@luzmo/analytics-components-kit/item-options-configs';
```

How to discover `option-key` values:
1. Pick an `item-type` from `@luzmo/analytics-components-kit/item-list` (or from the [Item Definitions](#available-item-types) list).
2. Open that itemŌĆÖs options config export from `@luzmo/analytics-components-kit/item-options-configs`.
3. Option keys are the `key` fields in the config (dot notation for nesting: `display.title`, `bars.roundedCorners`, ŌĆ”).

**State wiring**:
- Treat `.options` as app state (single source of truth).
- Listen to `luzmo-options-changed`, update your state, and pass the new options back into the component (one-way data flow).
- Render the configured chart with Luzmo Embed by passing `{ type, slots, options, filters }` (see [Start Here (End-to-End)](#start-here-end-to-end) and [Embedding with Luzmo Embed](#embedding-with-luzmo-embed)).

**Supported control types** (`customOptionConfiguration.control.type`):

**Note**: Control-specific fields differ per type. For the most accurate reference, inspect the `OptionConfig` / `OptionsConfig` types and/or the built-in item options configs under `@luzmo/analytics-components-kit/item-options-configs`.

**Filter Structure**:
```typescript
import type { ItemFilterGroup } from '@luzmo/analytics-components-kit/types';

// The `filters` property accepts an array of ItemFilterGroup (Luzmo filter format)
const filters: ItemFilterGroup[] = [
  {
    condition: 'and',
    filters: [
      {
        expression: '? in ?',
        parameters: [{ columnId: 'country-col-id', datasetId: 'dataset-id' }, ['Belgium', 'Netherlands']]
      }
    ],
    subGroups: [
      {
        condition: 'or',
        filters: [
          {
            expression: '? > ?',
            parameters: [{ columnId: 'amount-col-id', datasetId: 'dataset-id' }, 1000]
          }
        ]
      }
    ]
  }
];
```

**Multiple Filter Groups**:
When multiple ItemFilterGroups are passed, the UI treats them as: (group1) AND (group2) AND ...

```javascript
// Two separate root groups
const filters = [
  { condition: 'or', filters: [/* ... */], subGroups: [] },
  { condition: 'and', filters: [/* ... */], subGroups: [] }
];
```

**Example with Nested Groups**:
```javascript
const filters = [
  {
    condition: 'and',
    filters: [
      {
        expression: '? in ?',
        parameters: [
          { columnId: 'country-col-id', datasetId: 'dataset-id' },
          ['Belgium', 'Netherlands']
        ]
      },
    ],
    subGroups: [
      {
        condition: 'or',
        filters: [
          {
            expression: '? > ?',
            parameters: [{ columnId: 'date-col-id', datasetId: 'dataset-id' }, '2020-01-01T00:00:00Z']
          },
          {
            expression: '? > ?',
            parameters: [{ columnId: 'amount-col-id', datasetId: 'dataset-id' }, 1000]
          }
        ]
      }
    ]
  }
];
```

### Authentication & Configuration

Components that fetch data from the Luzmo API require authentication configuration. There are two ways to configure authentication:
Remember: embed authorization values come from a Luzmo authorization objectŌĆÖs `id` (key) and `token` (token) fields (or whatever names your backend maps them to).

#### 1. Per-Component Configuration (Recommended)
Pass embed authorization directly to components:

```html
<luzmo-droppable-slot
  api-url="https://api.luzmo.com"
  auth-key="your-auth-key"
  auth-token="your-auth-token">

```

#### 2. Global DataBroker Configuration
Configure the shared DataBroker instance once:

```typescript
import { dataBroker } from '@luzmo/analytics-components-kit/utils';

dataBroker.configure({
  apiUrl: 'https://api.luzmo.com',
  authKey: 'your-auth-key',
  authToken: 'your-auth-token',
  staleTime: 300000,      // Cache for 5 minutes
  gcTime: 600000,         // Garbage collect after 10 minutes
  retry: 3                // Retry failed requests 3 times
});
```

### DataBroker API

The DataBroker service handles all Luzmo API communication with built-in caching, retry logic, and error handling.

#### Key Methods

**fetchDatasetsWithColumnsAndFormulas(datasetIds: string[])**
Fetch datasets with their columns and formulas:

```typescript
const datasets = await dataBroker.fetchDatasetsWithColumnsAndFormulas(['ds_123', 'ds_456']);
```

**fetchDatasetsRecursive(datasetIds: string[])**
Fetch datasets and all linked datasets recursively:

```typescript
const allDatasets = await dataBroker.fetchDatasetsRecursive(['ds_123']);
const linkedIds = dataBroker.getLinkedDatasetsIds('ds_123');
```

**fetchData(query)**
Execute a data query:

```typescript
const result = await dataBroker.fetchData({
  dimensions: [
    { column_id: 'col_123', set_id: 'ds_456' }
  ],
  measures: [
    { 
      column_id: 'col_789', 
      set_id: 'ds_456',
      aggregation_function: 'sum'
    }
  ],
  where: [
    {
      column_id: 'col_123',
      set_id: 'ds_456',
      expression: 'equals',
      value: 'USA'
    }
  ],
  order: [
    { column_id: 'col_123', set_id: 'ds_456', direction: 'asc' }
  ],
  limit: {
    by: 100,
    offset: 0
  }
});

console.log(result.data);        // Data rows
console.log(result.performance); // Query performance metrics
```

**searchDatasets(query: string)**
Search for datasets:

```typescript
const datasets = await dataBroker.searchDatasets('sales');
```

**invalidateDatasetData(datasetId?: string)**
Invalidate cached data:

```typescript
dataBroker.invalidateDatasetData('ds_123');  // Invalidate specific dataset
dataBroker.invalidateAll();                   // Invalidate all cached data
```

### Caching & Performance

The DataBroker uses `@tanstack/query-core` for intelligent caching:

- **Automatic Caching**: API responses are cached by default (5 minutes stale time)
- **Deduplication**: Identical requests are deduplicated automatically
- **Background Refetch**: Stale data is refetched in the background
- **Network Resilience**: Automatic retry with exponential backoff
- **Offline Support**: Cached data remains available offline

```typescript
// Custom cache configuration
dataBroker.configure({
  staleTime: 10 * 60 * 1000,    // 10 minutes
  gcTime: 30 * 60 * 1000,        // 30 minutes
  retry: 5,
  retryDelay: (attempt) => Math.min(1000 * 2 ** attempt, 30000)
});
```

## Best Practices

### Import Best Practices
- **Always import individual components** - Only import what you need
- **Avoid importing all components** - Don't use catch-all imports
- **Tree-shaking friendly** - Individual imports allow bundlers to eliminate unused code
- **Better performance** - Smaller bundle sizes lead to faster loading times

### Data Wiring Patterns
- **Configure authentication once** - Use global DataBroker config when possible
- **Handle linked datasets** - Use `fetchDatasetsRecursive` to load joined datasets
- **Cache appropriately** - Adjust `staleTime` based on data freshness requirements
- **Handle errors gracefully** - Catch API errors and show user-friendly messages
- **Use TypeScript** - Leverage type definitions for better development experience

### Drag & Drop Patterns
- **Set drag boundaries** - Use `drag-boundary` to constrain dragging to specific areas
- **Provide visual feedback** - Components handle this automatically with `would-accept-drag` state
- **Handle drop events** - Listen to `luzmo-slot-contents-changed` to update your data model

### Language & Localization
- **Set language consistently** - Use same `language` prop across all components
- **App-wide locale control** - Import `setLocale` to set locale once at startup:
  ```ts
  import { setLocale } from '@luzmo/analytics-components-kit';
  setLocale('nl');
  ```
- **Support multi-language data** - Store labels as `{ en: 'Label', fr: '├ētiquette' }`
- **Use content-language** - Separate UI language from data language when needed
- **Available languages** - en, nl, fr, es, de

### Performance Patterns
- **Lazy load configurations** - Item definitions are loaded on-demand
- **Virtualize large lists** - Dataset selector uses virtualization automatically
- **Debounce search** - Search inputs have built-in debouncing
- **Optimize re-renders** - Use immutable updates for options and slots
- **Cache API responses** - DataBroker handles this automatically

### Accessibility
- All components support keyboard navigation
- Drag & drop has keyboard alternatives
- Form components work with screen readers
- Use proper ARIA labels

## Common Use Cases

### Tracking Slot Content Metadata

The `luzmo-info-visibility-changed` event fires when users toggle the info button in the slot configuration menu. This event provides access to the column/dataset IDs being displayed, which is useful for tracking, analytics, or synchronizing UI state.

```javascript
// Listen for info visibility changes
const droppableSlot = document.querySelector('luzmo-droppable-slot');

droppableSlot.addEventListener('luzmo-info-visibility-changed', (event) => {
  const { isVisible, slotContent } = event.detail;
  
  console.log('Info panel visibility:', isVisible);
  console.log('Dataset ID:', slotContent.datasetId || slotContent.set);
  console.log('Column ID:', slotContent.columnId || slotContent.column);
  console.log('Formula ID:', slotContent.formulaId || slotContent.formula);
  
  // Example: Track analytics
  if (isVisible) {
    analytics.track('slot_info_opened', {
      datasetId: slotContent.datasetId,
      columnId: slotContent.columnId
    });
  }
  
  // Example: Update external UI
  updateMetadataPanel({
    datasetId: slotContent.datasetId,
    columnId: slotContent.columnId,
    level: slotContent.level
  });
});
```

**Event Detail Structure**:
```typescript
{
  isVisible: boolean;           // Whether info IDs are now visible
  slotContent: {
    columnId?: string;          // Column ID (or use 'column')
    column?: string;            // Alternative column reference
    datasetId?: string;         // Dataset ID (or use 'set')
    set?: string;               // Alternative dataset reference
    formulaId?: string;         // Formula ID (if applicable)
    formula?: string;           // Alternative formula reference
    level?: number;             // Datetime/hierarchy level
    // ... other slot content properties
  }
}
```

#### Pattern 2: Dynamic Draggable Items (Manual Rendering)

For more control, you can manually create draggable data fields:

```typescript
async function loadDraggableFieldsForDatasets(datasetIds: string[]) {
  // Fetch datasets with columns and formulas
  const datasets = await dataBroker.fetchDatasetsWithColumnsAndFormulas(datasetIds);
  
  const columnsContainer = document.querySelector('.columns-panel');
  
  datasets.forEach(dataset => {
    // Render columns
    dataset.columns?.forEach(column => {
      const item = document.createElement('luzmo-draggable-data-field');
      item.data = {
        columnId: column.id,
        datasetId: dataset.id,
        type: column.type,
        subtype: column.subtype,
        label: column.name,
        description: column.description,
        lowestLevel: column.lowestLevel
      };
      item.language = 'en';
      item.size = 'm';
      
      columnsContainer.appendChild(item);
    });
    
    // Render formulas
    dataset.formulas?.forEach(formula => {
      const item = document.createElement('luzmo-draggable-data-field');
      item.data = {
        formulaId: formula.id,
        datasetId: dataset.id,
        type: formula.type,
        label: formula.name,
        description: formula.description
      };
      item.language = 'en';
      item.size = 'm';
      
      columnsContainer.appendChild(item);
    });
  });
}
```

#### Pattern 3: With Search and Filtering

Combine with Lucero components for search functionality:

```html
<div class="data-items-panel">
  <!-- Search input -->
  <luzmo-search id="search" placeholder="Search columns..."></luzmo-search>
  
  <!-- Filtered draggable items -->
  <div id="columnsList" class="columns-list"></div>
</div>
```

```typescript
type Column = { id: string; datasetId: string; type: string; name: Record<string, string> };

let allColumns: Column[] = [];
let filteredColumns: Column[] = [];

const search = document.getElementById('search') as HTMLElement;
const columnsList = document.getElementById('columnsList') as HTMLElement;

function renderColumns(columns: Column[]) {
  columnsList.replaceChildren(
    ...columns.map((column) => {
      const item = document.createElement(
        'luzmo-draggable-data-field'
      ) as HTMLElementTagNameMap['luzmo-draggable-data-field'];

      item.data = { columnId: column.id, datasetId: column.datasetId, type: column.type, label: column.name };
      item.language = 'en';
      item.size = 'm';

      return item;
    })
  );
}

function handleSearch(event: Event) {
  const searchTerm = (event as CustomEvent<{ value: string }>).detail.value.toLowerCase();
  
  filteredColumns = allColumns.filter(column => 
    Object.values(column.name).some(name => 
      name.toLowerCase().includes(searchTerm)
    )
  );

  renderColumns(filteredColumns);
}

search.addEventListener('value-changed', handleSearch as EventListener);
renderColumns(filteredColumns);
```

```ts
dashboardGrid.items = gridItems;

const closeEditPanelButton = document.getElementById('closeEditPanel') as HTMLButtonElement;
const editTabs = document.getElementById('editTabs') as HTMLElement;
const editTabData = document.getElementById('editTabData') as HTMLElement;
const editTabOptions = document.getElementById('editTabOptions') as HTMLElement;

const editMeasureSlot = document.getElementById('editMeasureSlot') as HTMLElementTagNameMap['luzmo-droppable-slot'];
const editItemPanel = document.getElementById('editItemPanel') as HTMLElementTagNameMap['luzmo-edit-item'];

  editTabData.hidden = tab !== 'data';
  editTabOptions.hidden = tab !== 'options';
};

const syncEditorsFromSelection = () => {
  const item = getSelectedItem();
  if (!item) return;

  editMeasureSlot.itemType = item.type;
  editMeasureSlot.slotsContents = item.slots ?? [];

  editItemPanel.itemType = item.type;
  editItemPanel.options = item.options ?? {};
  editItemPanel.slots = item.slots ?? [];
};

const openEditPanel = (tab: 'data' | 'options', itemId: string) => {
  selectedItemId = itemId;
  setEditTab(tab);
  editPanel.toggleAttribute('open', true);
  syncEditorsFromSelection();
};

const closeEditPanel = () => {
  selectedItemId = null;
  editPanel.toggleAttribute('open', false);
};

dashboardGrid.addEventListener('luzmo-grid-item-action', (e) => {
  const { action, id } = (e as CustomEvent<{ action: string; id: string }>).detail;
  if (action === 'edit-data') openEditPanel('data', id);
  if (action === 'edit-options') openEditPanel('options', id);
});

closeEditPanelButton.addEventListener('click', closeEditPanel);
editTabs.addEventListener('value-changed', (e) => setEditTab((e as CustomEvent<{ value: 'data' | 'options' }>).detail.value));

const setSlotContent = (slots: Array<{ name: string; content: unknown[] }>, name: string, content: unknown[]) =>
  slots.some((s) => s.name === name) ? slots.map((s) => (s.name === name ? { ...s, content } : s)) : [...slots, { name, content }];
```

## Wiring Configurations Safely

**ŌÜĀ’ĖÅ CRITICAL**: Improper wiring can cause infinite loops and crash the browser. Follow these patterns to avoid issues.

### The Problem: Infinite Update Loops

```javascript
// ŌØī WRONG - This will crash the browser!
function handleSlotChange(e) {
  slotsContents = e.detail.slotContents;
  // This triggers re-render, which updates luzmo-droppable-slot's property,
  // which triggers luzmo-slot-contents-changed again ŌåÆ INFINITE LOOP
}
```

### Solution 1: Use Event Flags

```javascript
let isInternalUpdate = false;

function handleSlotChange(e) {
  if (isInternalUpdate) return;  // Prevent circular updates
  
  isInternalUpdate = true;
  slotsContents = e.detail.slotContents;
  
  // Update other related state
  updateChartPreview();
  
  // Reset flag after update cycle completes
  setTimeout(() => { isInternalUpdate = false; }, 0);
}
```

### Solution 2: Deep Equality Check

```javascript
import { isEqual } from 'lodash-es';

function handleSlotChange(e) {
  const newSlots = e.detail.slotContents;
  
  // Only update if actually changed
  if (!isEqual(newSlots, slotsContents)) {
    slotsContents = newSlots;
    updateChartPreview();
  }
}
```

### Solution 3: One-Way Data Flow (Recommended)

```javascript
// State management (single source of truth)
const state = {
  chartType: 'bar-chart',
  slotsContents: [],
  chartOptions: {}
};

// Only update state in event handlers
function handleSlotChange(e) {
  // Update state
  state.slotsContents = e.detail.slotContents;
  
  // Render changes
  render();
}

function handleOptionsChange(e) {
  // Update state
  state.chartOptions = e.detail.options;
  
  // Render changes
  render();
}

// Render function applies state to components
function render() {
  // Pass state down to components (one direction only)
  const droppableSlot = document.querySelector('luzmo-droppable-slot');
  droppableSlot.slotsContents = state.slotsContents;
  
  const editItem = document.querySelector('luzmo-edit-item');
  editItem.options = state.chartOptions;
  editItem.slots = state.slotsContents;
  
  const vizItem = document.querySelector('luzmo-embed-viz-item');
  vizItem.item = {
    type: state.chartType,
    slots: state.slotsContents,
    options: state.chartOptions
  };
}
```

### Framework-Specific Solutions

**React** (with useState):
```jsx
function ChartBuilder() {
  const [slotsContents, setSlotsContents] = useState([]);
  const [chartOptions, setChartOptions] = useState({});
  
  // React handles re-render optimization automatically
  const handleSlotChange = (e) => {
    setSlotsContents(e.detail.slotContents);
  };
  
  const handleOptionsChange = (e) => {
    setChartOptions(e.detail.options);
  };
  
  return (
    <>
      <luzmo-droppable-slot
        slotsContents={slotsContents}
        onSlot-contents-changed={handleSlotChange}
      />
      <luzmo-edit-item
        options={chartOptions}
        slots={slotsContents}
        onOptions-changed={handleOptionsChange}
      />
      <luzmo-embed-viz-item
        item={{
          type: 'bar-chart',
          slots: slotsContents,
          options: chartOptions
        }}
      />
    </>
  );
}
```

### Best Practices Summary

Ō£ģ **DO**:
- Use a single source of truth for state
- Update state only in event handlers
- Let framework reactivity handle re-renders
- Use deep equality checks when needed
- Test with multiple rapid interactions

ŌØī **DON'T**:
- Update component properties directly in render functions
- Create circular data flows
- Mutate objects in place (use immutable updates)
- Forget to test performance with large datasets
- Assume synchronous updates (use callbacks/promises)

### Debugging Infinite Loops

If you encounter infinite loops:

1. **Add logging**:
```javascript
let updateCount = 0;
function handleSlotChange(e) {
  console.log('handleSlotChange called', ++updateCount);
  if (updateCount > 10) {
    console.error('Infinite loop detected!');
    debugger;  // Pause execution
    return;
  }
  // ... your code
}
```

2. **Check component properties**:
```javascript
// Before update
console.log('Before:', JSON.stringify(slotsContents));

// After update
slotsContents = e.detail.slotContents;
console.log('After:', JSON.stringify(slotsContents));

// Are they different?
```

3. **Use browser performance tools**:
- Chrome DevTools ŌåÆ Performance tab
- Record interaction
- Look for repeating function calls
